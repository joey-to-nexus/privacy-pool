핵심 방향
Privacy Layer를 기반으로 단계적으로 쌓는다.
게임 결제(Cross Pay)는 오프체인 결제를 온체인으로 가져오는 작업으로, Privacy Pool의 기본 Tx와 초기 사용량을 만드는 토대다.
Privacy Pool을 ERC-8004 Agent로 등록 → Identity Tree로 신원 인프라 → Compliance Layer로 규제 준수 → Expansion.
Compliance와 Identity는 같은 시점에 중요해진다. 두 레이어는 분리 개발하되 Phase 4에서 통합된다.

PHASE 1

Cross Pay

오프체인 결제를 온체인으로 가져온다. Privacy Pool의 기본 Tx와 초기 사용량 확보. Operator 없는 단일 단계 출금.
M2 · 1개월
Item Shop + Privacy Layer 결제 통합
ZK proof → claimWithdrawal 단일 단계 → 스텔스 주소에서 Item Shop 직접 ERC-20 수신.
Operator 없음. 2-stage 없음. IComplianceModule 연결 슬롯만 준비 (Phase 4 대비).
NEW
Gas 최적화
현재 HonkVerifier ~2.9M gas. Proof aggregation 또는 L2 환경에서의 calldata 압축으로 결제당 Gas 비용 절감. 목표: 반복 결제가 경제적으로 가능한 수준.
INFRA
Key 조회 API
Sequencer ECIES 인프라 재사용. 별도 컨트랙트 없음. 허용된 상대방에게만 npk + enc_pubkey 암호화 반환. Agent가 상대방 키를 자동 조회하여 결제 실행.
NEW
→ Game Agent: denomination 분해 → 입금 → ZK proof 생성 → claimWithdrawal → Item Shop 수령. Operator 개입 없음. 전 과정 자율 처리.

PHASE 2

Privacy Pool Agent

Privacy Pool을 ERC-8004 Agent로 등록. 다른 Agent에게 프라이버시 결제 인프라를 서비스로 제공.
M3 · 1개월
ERC-8004 Identity Registry 등록
서비스명 "privacy-preserving payment", 엔드포인트 Sequencer API, Pool fee 주소 등록.
Registration File: name / description / services(A2A, MCP) / agentWallet 구조 준수.
/.well-known/agent-registration.json 도메인 검증 파일 배포.
ERC-8004
Sequencer — Agent 위임 수신 인터페이스
외부 Agent로부터 결제 위임 요청을 수신하는 API 엔드포인트 추가.
요청: 수신자 키 + 금액 → Sequencer가 denomination 분해, 입금, ZK proof 생성, 출금 자율 처리.
Game Agent는 완료 이벤트(tx hash)만 수신.
NEW
ERC-8004 Reputation 초기 적립
처리 건수, 성공률, 응답 시간을 Reputation Registry에 기록 시작. Privacy Pool의 신뢰 지표를 온체인에 누적.
ERC-8004
→ 이 시점부터 ERC-8004 생태계의 다른 Agent들이 Privacy Pool을 자동 발견하고 결제를 위임할 수 있다.

PHASE 3

Identity Tree

Compliance가 요구되는 순간 이 모듈이 전제 조건이 된다. Phase 4 Compliance와 함께 통합.
M4 · 1개월
Identity ZK Circuit — Noir
Privacy Pool 회로와 동일 구조 (Poseidon2 + Merkle Inclusion).
증명: "이 신원 commitment가 유효한 Identity Tree에 존재한다."
Selective Disclosure: 공개할 속성만 public input.
KYC: isAdult / isKYC / sanctionsFree  |  KYA: agentId / controllerAddress / permissionScope
NEW
IdentityRegistry 컨트랙트
Issuer 서명 commitment → Merkle Tree 삽입 → Root 온체인 제출. IComplianceModule 구현체로 Phase 4에서 PrivacyPoolCore에 플러그인.
NEW
ERC-8004 Identity Backing
ERC-8004 agentId를 Privacy Pool Identity Tree의 commitment로 앵커링. ERC-8004 등록 사실을 ZK로 증명하되 agentId 자체는 비공개. Privacy Pool이 ERC-8004 신원을 신뢰 근거로 사용할 수 있는 연결 고리. ERC-8004 Draft 변경에 대응하기 위해 어댑터 레이어로 완전 격리.

MUST
IComplianceModule 구현 — IdentityRegistry 연결
Phase 3 IdentityRegistry를 IComplianceModule 구현체로 PrivacyPoolCore에 플러그인.
컨트랙트 재배포 없이 모듈 교체만으로 완료.
고액 출금, 규제 대상 거래에서 KYC / KYA ZK proof 요구.
ARCH
2-stage 출금 + AML 스크리닝
initiateWithdrawal → Operator 동시 검토 (≤5초) → attestWithdrawal.
OFAC 제재 주소 스크리닝. Controller 주소 기준 AML 체크. 스크리닝 결과 ECIES 암호화 감사 로그 보존.
24시간 후 자동 claimWithdrawal 가능 (검열 저항 유지).

PHASE 5

Expansion

Phase 1 상업적 검증 완료 후 시장 수요에 따라 우선순위를 결정. 방향성만 열어둔다.
M6+ · 시장 검증 후
⚠ 각 아이템의 착수 시점과 우선순위는 Phase 1 운영 데이터와 시장 반응을 기반으로 결정한다.
Privacy ENS
허용된 상대방에게만 키가 공개되는 선택적 이름 레지스트리. Key 조회 API의 온체인 확장.
Headless Agent Merchant
Agent가 서비스를 ERC-8004에 등록하고 자율 판매. 수익을 Privacy Pool에 자동 재예치.
Headless Agent Customer
Agent가 예산 범위 내에서 서비스를 자율 구매. Controller 승인 없이 소액 결제 실행.
Agent Intent Market
Agent가 Intent를 선언하면 Solver Agent들이 입찰. ERC-8004 Reputation으로 Solver 자동 선택.
A2A Advertising
광고주가 Pool에 예치 → Agent가 광고 소비 ZK proof 생성 → 보상 수령. 소비 내역 비공개.
